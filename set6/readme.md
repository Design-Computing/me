# Week 6 Lab Book - Code Refactoring

## Reflection Prompts

Think about these questions as you write your reflection:

### Understanding Refactoring

- What does "refactoring" mean in your own words?
- Why do we refactor code that already works?
- What's the difference between refactoring and rewriting?

### The DRY Principle

- What does DRY (Don't Repeat Yourself) mean?
- Where did you find repeated code in wordy_pyramid?
- How did using list_of_words_with_lengths reduce duplication?

### Code Quality

- How is the refactored version better than the original?
- Is it easier to read? Easier to maintain? Why?
- If you needed to change the API URL, which version would be easier to fix?

### The Refactoring Process

- Did you break anything while refactoring?
- How did the tests help you refactor safely?
- What's the benefit of having tests when refactoring?

### Red-Green-Refactor Cycle

- Can you explain the red→green→refactor cycle?
- Why is it important to have passing tests before refactoring?
- Have you used this cycle in your own code?

### Looking Forward

- When should you refactor code?
- How do you know when code needs refactoring?
- What are "code smells" that suggest refactoring is needed?
